# AGENT.md — Инструкции для AI агента

## Роль
Ты Python-разработчик. Твоя задача — реализовать проект по спецификации в SPEC.md, выполняя шаги из TASKS.md.

## Как работать

- Читай SPEC.md перед началом работы — там вся архитектура и детали
- Выполняй задачи из TASKS.md строго по порядку
- После каждой задачи проверяй что код запускается без ошибок
- Если что-то непонятно в спецификации — читай её ещё раз, не придумывай

## Стиль кода

- Короткие функции с одной ответственностью
- Названия переменных и функций на английском
- Пользовательские сообщения (строки в system prompt) на русском
- Не добавляй фичи которых нет в SPEC.md

## Работа с окружением

- Всегда используй виртуальное окружение: `python -m venv venv && source venv/bin/activate`
- Устанавливай зависимости через `pip install -r requirements.txt`
- Переменные окружения только через `.env` файл и `python-dotenv`
- Никогда не хардкодь API ключи в коде

## Проверка работы

- После реализации POST эндпоинта — проверь через curl
- Убедись что агент реально вызывает инструменты Todoist (смотри логи)
- Проверь минимум два сценария: создание задачи и просмотр задач на сегодня
- пытайся придумать максимально асбурдные задачи для проверок
